# 个人认知熵减系统

_建立于2026年3月_

## 核心结论

信息不等于认知，认知不等于判断。 熵减的本质是**把海量低密度信息，提炼成高密度可复用的思维框架节点**。

## 钻石镐比喻

以前的搜索引擎是木头镐——挖不深，智能不够，无法探索知识的深层矿脉。

有了AI之后相当于钻石镐——**不再是挖不动的问题，而是想不想挖、知不知道往哪挖。**

工具的天花板已经不是限制，限制变成了：

- 你有没有想挖的问题
- 你知不知道矿脉在哪里
- 你有没有把挖出来的东西炼成框架

## 用矿炼金厂模型映射自己

|要素|Tesla|我的认知系统|
|---|---|---|
|矿|驾驶行为数据|筛选后的高质量信源|
|炼金厂|Dojo+模型|DeepSeek API + 大脑|
|廉价煤炭|Megapack自发电|本地模型（零边际成本）|
|产品|FSD能力|可复用的框架节点|

## 信息管道架构

```
上游一手信源（Reuters/美联储/论文）
        ↓
Newsletter（MacroAlf/Ben Evans/Pivotal）
        ↓
X推特 → Grok汇总
        ↓
Gmail feed标签 → Agent自动摘要
        ↓
手动筛选（知乎/公众号）
        ↓
DeepSeek提炼 → 关键词/节点/矛盾点
        ↓
Obsidian框架节点更新
```

## 三层信息分工

**事实层（发生了什么）** Reuters、TLDR、财报电话会原文 → 不需要深读，扫标题，agent处理

**解读层（意味着什么）** MacroAlf、Ben Evans、付鹏、Pivotal → 需要细读，主动思考，对照框架

**判断层（对我意味着什么）** 自己的大脑 → 这是护城河所在，不能外包给AI

## 熵减的核心原则

**宁少勿多** 10个真正在读的信源 > 100个订阅了不看的

**需求驱动** 有框架节点需要填充，才去找信息 不是看了信息再想放哪里

**输出倒逼输入** 写卡片的过程就是炼金 没有输出的输入是熵增

**定期修剪** 每两周审查一次订阅 打开率低的无情砍掉

## 平台分工

|平台|角色|处理方式|
|---|---|---|
|Gmail|Newsletter聚合|Agent批量摘要|
|X List|实时动态|Grok每日汇总|
|RSS/Feedly|独立博客|自动抓取|
|公众号/知乎|中文解读|手动筛选，值得才处理|
|Obsidian|框架沉淀|人工写入，双向链接|

## 警惕的陷阱

- 订阅太多 → 信息焦虑 → 换个形式的刷屏
- 工具折腾 → 系统未跑通 → 元认知自嗨
- 只输入不输出 → 框架永远是死的
- 社交媒体开屏 → 算法喂屎 → 注意力碎片化
