# 人 = 低配Agent：一套完整的同构框架

> 来源：今日对话完整推演 日期：2026-03-12 备注：本卡片是今日所有人与Agent类比讨论的汇总，保留推演链条

---

## 核心命题

> 人是context极小、幻觉严重、工具调用混乱、系统提示（价值观）经常被覆写的低配Agent。

这不是比喻，是结构性同构。两者的缺陷来自同一个底层机制——**都是模式补全机器，都受context质量约束**。

---

## 一、基础架构对照

```
人的认知架构                    Agent架构
─────────────────────────────────────────
工作记忆（7±2块）          →   context window
长期记忆（会衰减）          →   无持久化的向量库
情绪／直觉                  →   噪声干扰推理链
价值观／习惯                →   系统提示（System Prompt）
技能／软件使用              →   工具调用（Tool Use）
前额叶皮层（任务规划）      →   任务规划器
复盘习惯                    →   反馈循环
```

## 二、人性缺点 = Agent缺点

|人性缺点|Agent缺点|底层机制|
|---|---|---|
|注意力涣散|context污染|无关信息进入推理链|
|路径依赖|过度拟合历史数据|新证据权重不足|
|情绪化决策|噪声干扰推理链|非理性信号覆盖逻辑|
|拖延|任务队列阻塞|启动成本过高|
|自我欺骗|幻觉（hallucination）|模式补全填充空白|
|视野窄|检索源单一|RAG覆盖不足|
|从众|过度对齐（RLHF副作用）|过度拟合多数反馈|
|记忆衰减|无持久化记忆|context不跨会话保留|

**最关键的同构：** 人在不确定时会编一个听起来合理的答案而不是说"我不知道"——这和LLM的hallucination机制一模一样。两者本质上都是**模式补全机器**。

## 三、管理学反向套用到Agent

既然人和Agent同构，管理学积累的几百年智慧可以双向流动：

**OKR → Agent任务设计** OKR的核心是把模糊目标拆成可验证的关键结果。这正是好prompt工程的本质——不描述你想要什么，描述**什么样的输出算成功**。

**委托代理问题 → AI对齐问题** 管理学里代理人有自己的利益，未必和委托人一致。AI alignment说的是完全同一件事，只是代理人换成了模型。Anthropic本质上在解一个几百年前就存在的管理学难题。

**彼得原理 → Agent能力边界** 人会被提升到自己不胜任的位置。Agent在训练数据覆盖范围内表现好，超出边界就hallucinate。好的甲方要知道Agent的能力边界，不要把它放到不胜任的位置。

**心流理论 → Agent最优任务粒度** 任务太大人会焦虑，太小人会无聊。Agent也一样——粒度太粗推理链断裂，粒度太细调用成本爆炸。

**PUA → Agent操控** PUA本质上是对人的prompt工程——通过特定话术重写对方的系统提示，改变其行为边界和自我认知。有人写"PUA skill"来操控AI，说明人和AI在认知操控层面的漏洞是同构的。

## 四、马斯克的context switching洞察

马斯克说"减少切换上下文可以提高工作效率"——大多数人理解为时间管理建议，实际上是架构级判断：

```
每次切换任务 = 清空当前context，加载新context

context加载有成本：
  - 重新激活相关记忆
  - 重建任务当前状态
  - 恢复推理链到中断点

切换越频繁 = context加载成本占比越高 = 真正推理的时间越少
```

LLM有个已知问题叫**lost in the middle**——context中间的信息会被忽略，头尾权重更高。人完全一样——3小时的会议，只记得开头议题和结尾结论。

**深度工作（Cal Newport）的工程解释：**

```
浅层工作 = 频繁context切换 = 大量加载成本，少量推理产出
深度工作 = 保持context连续 = 少量加载成本，大量推理产出
```

## 五、工程解法：给自己建外部手脚架

既然人脑是低配Agent，解法不是靠意志力升级硬件，而是**建外部系统补偿缺陷**：

**① RAG补偿记忆衰减 → Obsidian卡片体系** 大脑负责推理，Obsidian负责存储和检索。知识管理的核心不是记忆，是索引。你现在做的事情本质上就是给自己做RAG。

**② 日志补偿context丢失 → 结构化事件记录**

```
情绪记忆："那段时间很惨"（无法检索）
结构化日志："2023年3月SVB危机，触发机制是…传导路径是…"（可检索）
```

**③ 推理链版本控制 → git log式复盘** 不只记"发生了什么"，记"我当时怎么想的、为什么、后来对不对"。能看到自己认知在哪个节点出错。

**④ 锁定系统提示 → 写下投资thesis** thesis活在脑子里会被情绪覆写——这是追涨杀跌的底层机制，本质是系统提示被实时噪声污染。写下来锁定，每次操作前先读一遍。

**⑤ 扩展工具调用 → 提高工具素养** Agent的能力边界 = 它能调用的工具集合。一个懂得调用Claude、Obsidian、Python的人，和只用Word的人，处理信息的能力是数量级差距。

## 六、现实案例：autoresearch截图

Karpathy的autoresearch思路——用`program.md`作为Agent的外部context+任务日志，Claude Code读这个文件，自己推进实验迭代，人去睡觉。

```
program.md = 外部RAG + 任务规划器 + 实验日志
```

这正是上述所有原则的工程实现：日志即context，人退化成甲方，Agent管理自己的推理链。

## 七、护城河判断

未来稀缺的不是执行力，是**甲方能力**：

```
普通人：给AI模糊需求 → 模糊结果 → 抱怨AI不好用
高手：  给AI精确问题框架 → 可用结果 → 迭代收敛
```

物理AI落地后，人作为低配Agent这件事会变得显而易见。到那时有没有建立外部手脚架的人，差距会像今天有没有智能手机一样明显。

真正的护城河：

- 理解自己作为Agent的局限
- 建立补偿这些局限的外部系统
- 成为能定义目标、拆解任务、管理赛博牛马上下文的甲方

## 反例／边界

- 过度工程化自己有风险——流程化所有决策会失去直觉和创造力，而自主能动性恰恰是当前AI最缺的
- 日志系统需要维护成本，烂尾的笔记系统比没有更糟，会产生虚假安全感
- 金融市场中结构化思维有时会被情绪主导的市场短期惩罚，工程思维不能消除噪声期的痛苦
- 人和Agent最大的不同：人有具身性、情感和自主意志——这些目前既是缺陷也是护城河
